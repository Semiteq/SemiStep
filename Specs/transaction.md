# Transaction protocol over Modbus TCP (concept)

## 1) Header

Header содержит общую информацию о протоколе и адреса областей.

Поля `int16/uint16` можно читать без знания word order.

| Id | Имя                    | Тип    | Описание |
|----|------------------------|--------|----------|
| h1 | magic_number           | uint16 | Константа для обнаружения протокола (например 69) |
| h2 | word_order_32          | uint16 | Порядок слов для 32-бит полей: `1 = high-word first`, `0 = low-word first` |
| h3 | protocol_version       | uint16 | Версия протокола/разметки памяти. Сейчас `1` |

Далее адреса областей (32-бит, интерпретируются с учётом `word_order_32`):

| Id | Имя                    | Тип    | Описание |
|----|------------------------|--------|----------|
| h4 | managing_area_address  | uint32 | Адрес (0-based offset holding registers) первого регистра области `Managing area` |
| h5 | table_info_address     | uint32 | Адрес (0-based offset holding registers) первого регистра области `Table info` |

h1 - абсолютный адрес, задаем в приложении, все отсчеты далее от него, через offset.

## 3) Managing area

`Managing area` — область для координации массовой записи и явного commit.

| Id | Имя               | Тип    | Кто пишет | Описание |
|----|-------------------|--------|-----------|----------|
| m1 | updating          | uint16 | Клиент    | `1 = идёт запись`, `0 = таблица активна/свободна` |
| m2 | session_id        | uint32 | Клиент    | Идентификатор текущей транзакции (один на всю bulk-операцию) |
| m3 | commit_request_id | uint32 | Клиент    | Запрос commit. Клиент пишет сюда значение `session_id`, когда считает, что все чанки записаны |
| m4 | committed_id      | uint32 | ПЛК       | Подтверждение commit. ПЛК пишет сюда значение `commit_request_id`, когда commit принят |
| m5 | error_code        | uint16 | ПЛК       | Код ошибки (0 = нет ошибки) |

## 4) ? CRC area

[Doc1](https://docs.tia.siemens.cloud/r/en-us/v20/extended-instructions-s7-1200-s7-1500/diagnostics-s7-1200-s7-1500/getchecksum-read-out-checksum-s7-1200-s7-1500) - неизвестна стоимость операции crc и можно ли считать от произвольного диапазона в памяти, а не от конкретного объекта.

| Id | Имя               | Тип    | Кто пишет | Описание |
|----|-------------------|--------|-----------|----------|
| c1 | request_crc       | uint16 | Клиент    | Запрос для ПЛК посчитать хэш всей таблицы |
| c2 | crc_start         | uint16 | Клиент    | Начало диапазона для расчета crc |
| c3 | crc_length        | uint16 | Клиент    | Длина диапазона для расчета crc |
| c4 | crc_result        | uint32 | ПЛК       | Поле для записи результата расчета crc |

## 5) Table info

| Id | Имя             | Тип   | Описание |
|----|-----------------|-------|----------|
| t1 | column_count    | int16 | Number of columns |
| .. | Table info data | ...   | ... |

Составной тип данных для `Table Info`:

| Тип    | Имя           | Описание |
|--------|---------------|----------|
| uint16 | separator     | Пустой разделить 0x16 |
| uint16 | column_number | |
| string | column_name   | ASCII имя столбца |
| uint16 | data_type     | 0 - int16, 1 - int32, 2 - uint16, 3 - uint32, 4 - real, 5 - string |
? | uint16 | column_type   | 0 - combobox, 1 - значащий тип |
| real   | default_value | Если тип не совпадает, будет неявно приведен |
| real   | min_value     | если тип не совпадает - будет неявно приведено
| real   | max_value     | если тип не совпадает - будет неявно приведено |
bool - always positive
| bool   | using_min     | Флаг использования значения по умолчанию |
| bool   | using_max     | Флаг использования значения по умолчанию |

Тип данных повторяется для каждого столбца.

## Definitions

**Регистр** — 16-бит holding register (Modbus).

**Word order (32-bit)** — порядок двух 16-бит слов при представлении 32-бит значений (uint32/int32/float32):
- `1 = high-word first` (сначала старшее слово, затем младшее)
- `0 = low-word first` (сначала младшее, затем старшее)

**Чанк** — набор подряд идущих holding registers, записываемый за один запрос FC16 (Write Multiple Registers). Максимум 123 регистра на запрос (ограничение протокола).

**Транзакция / bulk update** — логическая операция массовой записи данных (например загрузка таблицы из файла, вставка строки со сдвигом и т.д.), состоящая из нескольких чанков.
